# Notifications & Dialogs Specification

**Status**: Draft
**Created**: 2026-01-19
**Owner**: Eric

## 背景与目标
弹窗与通知遍布各域，导致重复与样式不一致。目标是统一通知与对话框的触发与渲染入口。

## 范围（In Scope）
- 应用级通知与提示的统一触发接口。
- 弹窗与确认框的集中式管理。
- 通知与对话框的优先级与排队规则。

## 非目标（Out of Scope）
- 各业务域的具体文案与业务判断。

## 用户故事
- 用户看到一致风格的通知与弹窗。
- 业务模块可通过统一入口触发提示。

## 验收标准
- UI 只通过通知/对话框服务触发提示。
- 通知状态可追踪并支持自动清理。

## 约束与假设
- 通知与弹窗组件集中在 UI 层公共模块。

## 开放问题
无。
